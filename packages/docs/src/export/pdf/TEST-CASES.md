# PDF 导出测试用例 — 边界情况覆盖

用于本地手动测试 PDF 导出功能。在编辑器中创建以下内容的文档，逐一导出并验证。

---

## 测试 1：空文档

**输入：** 打开一个全空的文档（无任何内容），点击"导出为 PDF"

**预期：**
- 不报错，成功下载 PDF
- PDF 只有一页空白（或仅有页边距）

---

## 测试 2：纯中文长段落（CJK 换行）

**输入：**
```
这是一段非常长的中文文本用来测试中文换行是否正常工作因为中文之间没有空格所以如果分词逻辑按空格切分的话整段文本会被当成一个单词从而导致溢出右边距或者整行超出页面宽度这个问题在实际使用中会非常明显因为中文文档是主要使用场景我们需要确保每一行都能正确断行不超出页面边界同时保持良好的阅读体验文字应该在接近右边距时自动换到下一行而不是一直延伸到页面外面去
```

**预期：**
- 文本应在页面右边距处自动换行
- 不应有文字超出页面边界
- 字体显示正常（不是方块）

---

## 测试 3：中英文混合 + 所有 inline mark

**输入：**
```
这是 **粗体中文** 和 *斜体英文* 以及 `行内代码 code`。
还有 ~~删除线~~ 和 [链接文字](https://example.com) 以及 下划线文字。
混合：**这是粗体 bold 混合中英文** 再加一个 $E=mc^2$ 公式。
```

**预期：**
- 粗体、斜体、代码、删除线、链接、下划线各自正确渲染
- 中文粗体：如果字体限制无法真正加粗，至少不报错
- 链接有蓝色 + 下划线
- 删除线有中间横线

---

## 测试 4：多级嵌套列表

**输入：**
```
- 第一层 item 1
  - 第二层 item 1.1
    - 第三层 item 1.1.1
      - 第四层 item 1.1.1.1
  - 第二层 item 1.2
- 第一层 item 2

1. 有序第一层
   a. 有序第二层
      i. 有序第三层
2. 有序第一层 item 2
```

**预期：**
- 每层缩进递增
- bullet 符号按层级变化（•, ◦, ▪, ‣）
- 有序列表编号格式按层级变化（数字、字母、罗马）

## 测试 30：混合内容跨页一致性

**输入：** 一个文档按顺序包含：H1 → 段落 → 表格 → 图片 → 代码块 → 列表 → 引用 → callout

且每个块的长度刚好在页面底部触发换页

**预期：**
- 每个块换页后接续正常
- 不出现内容重叠
- 段间距一致

## 已知限制（待记录实际表现）

| 场景 | 预期表现 | 严重程度 |
|------|----------|----------|
| 中文长段落不换行 | 文字超出右边距 | 🔴 高 |
| Blockquote 跨页竖线 | 竖线画到错误坐标 | 🟡 中 |
| renderSegments 跨页返回值 | ctx.y 坐标倒退，后续内容位置错乱 | 🔴 高 |
| 大图 base64 转换 | GC 压力大 + 可能卡顿 | 🟡 中 |
| 表格 cell 内 mark 丢失 | 粗体/链接不显示 | 🟡 中（仅纯文本） |
| 中文粗体视觉无区分 | 看起来和正文一样 | 🟢 低 |
| 文件名特殊字符 | 可能下载失败或乱码 | 🟡 中 |
